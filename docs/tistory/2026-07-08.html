AI 시대에 취미가 곧 비즈니스 기회가 되는 이유

LLM이 지식 노동을 평준화하면서 경쟁 우위는 '도메인 지식'으로 이동했습니다.

개인 맥락 데이터는 LLM 간에 쉽게 마이그레이션되므로 경쟁 우위가 될 수 없습니다.

진정한 방어력은 언어화하기 어려운 암묵지와 인간 의존도가 높은 분야에서 나옵니다. (예: Karrot=물리적 관계, Anduril=방산)

취미는 결과가 아니라 과정이 목적이며, 실행 주체가 '나'여야만 가치가 발생하는 체득형·물리적 활동입니다. 이는 휴머노이드/AI가 대체할 수 없습니다. 오히려 AI는 노동을 줄여 취미 시간을 늘려줍니다. '배우기 쉽지만 마스터하기 어려운' 구조이므로 진입 장벽을 낮추는 것이 곧 비즈니스 기회가 됩니다.

AI가 보완재인 단계에서는 노동 시간이 오히려 늘어납니다(NBER에 따르면 주당 3.15시간 증가).

하지만 개인 영역이나 산출이 고정된 업무에서는 절약된 시간이 여가로 이어집니다. 변곡점은 AI가 대체재(에이전트/휴머노이드)로 넘어가는 순간입니다.

여가 지출 비중은 이미 9.5%에서 13%로 상승했습니다(Visa).

비즈니스 3가지 패턴

커뮤니티가 해자 — Strava (기록 인정·경쟁 허브)

습관이 반복 매출 — Chess.com (월 구독 AI 코치)

도메인 지식이 AI 워크플로우 — GOATY (골프, 진단→추적 루프), Garmin (센서 하드웨어 계층 소유)

Show GN: gh-attach – CLI로 GitHub 이슈/PR에 이미지, 파일 첨부하기

GitHub 이슈나 PR에 이미지를 첨부하려면 웹 UI에 직접 접속해서 업로드해야 했습니다. 첨부 파일 업로드를 지원하는 공식 API가 없기 때문입니다.

gh-attach는 로컬 브라우저에 저장된 로그인 쿠키를 읽어 웹과 동일한 업로드 API를 호출하는 gh CLI 확장입니다.

$ gh extension install sudosubin/gh-attach $ gh attach ./image.png -R owner/repo # https://github.com/user-attachments/assets/...

Chrome, Firefox, Safari 등 주요 브라우저 쿠키 지원 (Profile/Container 지정 가능)

로그인된 GitHub 계정과 일치하는 세션을 자동 선택

--json, --jq, --template 옵션으로 Script에 연동 가능

Go로 작성, gh 확장 또는 standalone 바이너리(Homebrew/GitHub)로 배포

스크린샷을 매번 수동으로 첨부하기 귀찮아서 만들었는데, 에이전트가 PR 생성 시 다이어그램, 스크린샷을 자동으로 업로드하게 할 때도 유용합니다.

3. GeekNews

Show GN: 대본(YAML) 한 장이면 나레이션·자막·BGM 입힌 영상이 나오는 도구를 만들었습니다

영상 편집이 처음이라 한 편에 반나절씩 걸려서, 편집 과정을 코드로 자동화했습니다.

입력은 장면별로 narration / visual / callout을 적은 YAML 한 장이 전부입니다. python3 src/pipeline.py scenario.yaml config.yaml 한 줄이면 완성된 mp4가 나옵니다.

파이프라인 (Python):

대본 → TTS 나레이션 (say / Supertone / ElevenLabs 교체식, 결과 캐싱)

나레이션 길이에 맞춰 자막 타이밍 자동 배분, 내레이션=자막 동일

자막·콜아웃을 PNG로 렌더해 overlay (이 ffmpeg 빌드에 libass/drawtext가 없어 택한 방식 — 한글 외곽선·위치 픽셀 제어가 오히려 쉬움)

세로로 찍은 폰 영상은 배경 블러 패딩으로 화면비 자동 보정 (config 한 줄로 16:9 ↔ 9:16 쇼츠)

나레이션 concat + BGM 믹스 (영상 끝에 맞춰 BGM 자동 컷 + 페이드 → '검은 꼬리' 방지)

렌더 끝에 길이 드리프트·마지막 프레임 밝기 자동 검사

faceless 자막 영상 전용이고, 실제로 이 도구로 제 유튜브 채널 영상을 뽑고 있습니다.

한계: 자막이 단어 단위 싱크가 아니라 장면 단위(글자수 비례 배분)입니다. 그리고 이 환경 ffmpeg에 libass가 없다는 전제로 만들어서, PNG overlay 방식이 기본입니다.

완벽한 편집기 대체가 아니라, '대본 → 초벌 영상'을 사실상 0초로 만들어 반복 제작 비용을 없애는 게 목적입니다.

스모크 테스트(키 없이 macOS say 음성으로 전체 파이프라인 확인): python3 -m pip install -r requirements.txt python3 demo/make_placeholders.py python3 src/pipeline.py demo/scenario.yaml demo/config.yaml

오늘의 정처기 문제

소프트웨어 개발 방법론 중, 요구사항 변경에 유연하게 대응하고 짧은 주기로 반복적인 개발을 통해 점진적으로 결과물을 만들어내는 접근 방식은?

  1. 폭포수 모델 (Waterfall Model)
  2. 나선형 모델 (Spiral Model)
  3. 정답 · 애자일 모델 (Agile Model)
  4. V 모델 (V-Model)

해설 애자일 모델은 변화하는 요구사항에 유연하게 대응하기 위해 짧은 개발 주기를 반복하고, 고객과의 지속적인 소통을 통해 점진적으로 소프트웨어를 개발합니다. 반면 폭포수 모델은 각 단계를 순차적으로 진행하며, V 모델은 테스트를 각 개발 단계와 연결하고, 나선형 모델은 위험 관리에 중점을 둡니다.

오늘의 IT · 개발 · 기획 용어

  • 멱등성 (Idempotence) 개발
    동일한 연산을 여러 번 수행하더라도 항상 동일한 결과를 반환하며, 시스템의 상태를 변경하지 않는 특성입니다.
  • 서킷 브레이커 (Circuit Breaker) 개발
    분산 시스템에서 장애가 발생한 서비스로의 요청을 일시적으로 차단하여 시스템 전체의 연쇄적인 장애를 방지하는 디자인 패턴입니다.
  • 온보딩 (Onboarding) 기획
    새로운 사용자나 직원이 제품이나 서비스, 또는 조직에 성공적으로 적응하도록 돕는 일련의 과정입니다.
