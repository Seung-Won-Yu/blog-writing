Show GN: 러닝 대회가 답답해서 직접 만든 러닝앱 "LLRUN" (Android/iOS)

달리기는 공짜인데 "대회"는 선착순 마감에 참가비 10~15만원이라 직접 만든 러닝앱입니다. 1인 개발, 무료입니다.

핵심 아이디어: 특정 날짜·장소에 모여야 대회가 되는 게 아니라, 코스가 등록돼 있으면 아무 때나 대회 모드로 달릴 수 있게.

챌린지런: 코스별 기록 경쟁 (개인 PB / 코스 순위)

경로비행: 달린 GPS 경로를 위성지도 위 3D 비행 영상으로 재생

아트런: GPS 폴리라인으로 지도에 그림 그리기

크루: 크루 누적거리 랭킹, 크루 대항전

기술 스택: React 18 + Capacitor(안드로이드/iOS 동시), 지도 MapLibre GL, 백엔드 Spring Boot. 걸음은 안드로이드 TYPE_STEP_COUNTER를 포그라운드 서비스로 직접 처리하고, 백그라운드 유실을 막기 위해 앱 종료 시에도 서비스가 센서를 붙잡는 방식으로 구현했습니다. 갤럭시워치 러닝 걸음은 워치 연동으로 별도 반영합니다.

다운로드: https://www.finiteorder.com/download

만들면서 겪은 것들(자정 넘은 걸음수 롤오버 버그, GPS 부정행위 검증 등) 질문 주시면 답변 드리겠습니다.

3. GeekNews

sem - Git 위에 함수/클래스 단위 변경 추적을 추가하는 시맨틱 버전 관리 도구

sem은 Git 위에 함수/클래스 단위 변경 추적 기능을 추가하는 시맨틱 버전 관리 도구입니다. 이 도구는 줄 단위가 아닌 코드 엔티티를 비교하여 어떤 함수, 메서드, 클래스가 추가, 수정, 삭제, 이동, 이름 변경되었는지 보여줍니다. 예를 들어, “line x~y가 변경됨” 대신 “함수 blah가 변경됨”과 같이 구체적인 정보를 제공합니다.

sem은 별도의 설정 없이 모든 Git 저장소에서 작동하며, Git 외부에서도 임의의 파일을 비교할 수 있습니다. 엔티티 캐시는 SQLite에 저장되며, 저장소 외부의 OS 캐시 디렉토리에 보관되어 작업 트리를 오염시키지 않습니다. tree-sitter를 사용하여 코드를 파싱하고, 엔티티 ID, AST 구조 해시, 80% 초과 토큰 유사도 순서로 변경 전후 코드를 연결하여 이름 변경 및 이동을 감지합니다.

지원 명령들

sem diff: 작업 트리, 스테이징, 커밋, 커밋 범위, 임의의 두 파일을 비교하며, 일반 텍스트, Markdown, JSON 출력을 지원합니다.

sem impact: 특정 함수 변경 시 영향을 받는 의존 코드, 호출자, 테스트를 찾아줍니다.

sem blame: 함수, 클래스, 메서드의 엔티티 레벨 마지막 수정자를 보여줍니다.

sem log: Git 히스토리를 통해 하나의 엔티티가 어떻게 변화했는지 추적합니다. 엔티티를 지정하지 않으면 저장소의 핫스팟(자주 변경된 부분)과 동시 변경 쌍(반복적으로 함께 변경된 부분)을 분석하여 “하나를 건드리면 이것도 잊지 마세요”와 같은 정보를 제공합니다.

sem entities: 현재 폴더 아래의 모든 엔티티를 표시합니다.

sem context: 에이전트에 필요한 주변 코드만 토큰 예산에 맞춰 구성해줍니다.

sem setup: 실행하면 기존 git diff가 엔티티 단위 출력으로 변경되고, 스테이징된 변경의 영향 범위를 보여주는 pre-commit 훅이 설치됩니다. macOS와 Linux에서는 Claude Code에 상주 의존 그래프와 프롬프트 시작 시 컨텍스트 훅을 연결하여 매번 저장소를 다시 분석하거나 코드를 찾아다니는 비용을 줄입니다.

sem mcp: diff, impact, context, entities, blame, log를 MCP 도구로 제공하여 코딩 에이전트가 전체 파일 대신 필요한 함수와 의존 관계만 조회하도록 합니다.

GitHub Action을 추가하면 PR마다 변경된 함수, 클래스, 메서드를 하나의 고정 댓글로 갱신하며, 포맷이나 주석만 바뀐 PR도 구분합니다. TypeScript, Python, Go, Rust, Java, Swift, SQL, C/C++ 등 32개 언어와 JSON, YAML, TOML, CSV, Markdown을 지원하며, 미지원 파일은 청크 단위 diff로 처리합니다. 라이선스는 MIT 또는 Apache-2.0입니다.

오늘의 정처기 문제

소프트웨어 개발 과정에서 품질을 보증하고 위험을 관리하기 위해, 개발 초기부터 전체 생명주기 동안 체계적인 활동을 수행하는 것은?

  1. 테스트 자동화 (Test Automation)
  2. 형상 관리 (Configuration Management)
  3. 정답 · 소프트웨어 품질 보증 (Software Quality Assurance, SQA)
  4. 애자일 방법론 (Agile Methodology)

해설 소프트웨어 품질 보증(SQA)은 소프트웨어 제품 및 개발 프로세스의 품질을 확보하기 위한 체계적인 활동입니다. 이는 개발 초기부터 전체 생명주기에 걸쳐 오류를 예방하고 품질 표준을 준수하도록 관리합니다. 테스트 자동화는 SQA의 한 부분이며, 형상 관리는 변경 사항을 관리하고, 애자일 방법론은 개발 접근 방식 중 하나입니다.

오늘의 IT · 개발 · 기획 용어

  • 데이터 마트 (Data Mart) IT
    데이터 웨어하우스에서 특정 부서나 사용자의 요구에 맞춰 정제된 데이터를 담은 작은 규모의 데이터 저장소입니다.
  • SSR (Server-Side Rendering) 개발
    웹 페이지를 서버에서 미리 렌더링하여 클라이언트에 전송하는 방식으로, 초기 로딩 속도 개선과 SEO에 유리합니다.
  • 퍼스트 파티 데이터 (First-Party Data) 기획
    기업이 자사 웹사이트, 앱, CRM 시스템 등을 통해 직접 수집한 고객 데이터로, 신뢰도가 높고 활용 가치가 큽니다.

이 글은 공개 뉴스 페이지를 기반으로 자동 수집·요약한 학습용 다이제스트입니다. 수치, 정책, 제품 정보처럼 정확성이 중요한 내용은 반드시 원문을 함께 확인해 주세요.
