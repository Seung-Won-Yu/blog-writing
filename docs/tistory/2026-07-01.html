  • 서비스의 가용성을 높이고 사용자 임팩트를 줄이는 방법은 여러 가지가 있지만, 이번 작업은 Alert 시스템 개선에 집중했습니다. Alert은 장애 예방과 장애 대응 사이를 잇는 운영 인터페이스에 가깝습니다. 위험 신호를 더 빨리 발견하면 실제 장애로 이어지기 전에 조치할 수 있고, 장애가 발생한 뒤에도 담당자가 더 빠르게 인지하고 대응을 시작할 수 있습니다.

오늘의 정처기 문제

운영체제에서 메모리 할당 전략 중, 프로세스가 필요로 하는 만큼의 메모리를 처음부터 모두 할당하는 방식은?

  1. 최초 적합 (First Fit)
  2. 최적 적합 (Best Fit)
  3. 최악 적합 (Worst Fit)
  4. 완전 적합 (Entire Fit)

해설 완전 적합(Entire Fit) 방식은 프로세스 실행에 필요한 모든 메모리를 시작 시점에 한 번에 할당합니다. 이는 메모리 단편화를 줄일 수 있지만, 프로세스가 실제로 모든 메모리를 사용하지 않을 경우 낭비가 발생할 수 있습니다.

오늘의 IT · 개발 · 기획 용어

  • CQRS (Command Query Responsibility Segregation) 개발
    명령(데이터 변경)과 질의(데이터 조회)의 책임을 분리하여 서로 다른 모델이나 서비스로 구현하는 아키텍처 패턴입니다.
  • DNS 라운드로빈 (DNS Round Robin) IT
    하나의 도메인 이름에 대해 여러 IP 주소를 등록하고, 요청이 올 때마다 등록된 IP 주소들을 순차적으로 응답하여 트래픽을 분산하는 방식입니다.
  • 프라이싱 (Pricing) 기획
    제품이나 서비스의 가격을 결정하는 전략 및 과정으로, 시장 상황, 경쟁사, 원가, 고객 가치 등을 고려합니다.
