AI 코딩도 결국 ‘자주 누르는 버튼’이 생겼다

OpenAI가 예고한 Codex Micro는 AI 코딩이 채팅창을 넘어 물리 버튼으로 이동하는 장면을 보여줍니다. databow는 여러 데이터베이스를 하나의 CLI 흐름으로 다루게 합니다. 둘은 전혀 다른 제품처럼 보이지만, 반복 작업을 짧고 예측 가능한 인터페이스로 바꾼다는 공통점이 있습니다. 다만 티저 단계의 추정과 실제 공개 기능은 구분해서 봐야 합니다.

먼저 보는 결론

  • Codex Micro: 확정된 제품 기능보다 AI 작업을 물리 단축키로 다루려는 방향이 흥미롭습니다.
  • databow: ADBC를 이용해 SQL 실행과 결과 내보내기를 일관된 CLI로 묶습니다.
  • 공통점: 좋은 도구는 새로운 기능을 늘리기보다 반복 동작의 마찰을 줄입니다.

무엇이 사실이고 무엇이 추정인가

Codex Micro는 티저에 등장한 형태와 공개일이 확인된 상태이고, 구체적인 버튼 기능·IDE 연동·커스터마이즈 범위는 정식 발표 전 추정입니다. 반면 databow는 지원 명령과 출력 형식이 공개된 실행 가능한 도구입니다. 기사에서 ‘보였다’와 ‘지원한다’를 구분하면 기대를 과장하지 않고 읽을 수 있습니다.

databow - ADBC로 데이터베이스를 조회하는 CLI 도구

ADBC(Arrow Database Connectivity)를 통해 데이터베이스에 연결하고 SQL을 실행하는 커맨드라인 도구입니다.

호환되는 ADBC 드라이버가 있는 모든 데이터베이스에 연결할 수 있습니다.

명령 히스토리와 직관적인 네비게이션을 지원하는 대화형 SQL 셸을 제공하며, SQL 쿼리 하이라이팅 기능도 있습니다.

동적인 컬럼 너비로 깔끔하게 정렬된 테이블 형태로 결과를 표시하고, 쿼리 결과를 JSON, CSV, Arrow IPC 파일로 익스포트할 수 있습니다.

대화형 사용 외에도 `--query`로 쿼리를 직접 실행하거나, stdin 입력 또는 `--file`로 파일을 실행하고 `--output`으로 결과 파일을 저장하는 등 비대화형 사용도 지원합니다.

Apache-2.0 라이선스로 Rust로 구현되었습니다.

ADBC(Arrow Database Connectivity)는 Apache Arrow 프로젝트의 하위 표준으로, 서로 다른 데이터베이스에서 Arrow 데이터를 넣고 빼기 위한 단일 API입니다.

JDBC/ODBC의 컬럼 지향 대안으로, 결과를 행(row) 단위가 아닌 컬럼 단위 Arrow 데이터로 반환합니다. 이는 대량 컬럼 분석 워크플로우에서 행 지향 형식으로 변환했다가 되돌리는 비용을 피할 수 있어 ODBC/JDBC 대비 효율적입니다.

2023년에 1.0.0이 발표되었고 현재 버전은 1.1.0입니다.

지원하는 데이터베이스(드라이버)로는 공식적으로 PostgreSQL, SQLite, DuckDB, Snowflake, BigQuery, Flight SQL 지원 DB가 있으며, ADBC Driver Foundry 확장을 통해 Amazon Redshift, Apache DataFusion, Apache Spark, ClickHouse, Databricks, Exasol, Microsoft SQL Server도 지원합니다.

OpenAI, Codex 전용 미니 키보드 ‘Codex Micro’ 티저 공개

왜 궁금한가: AI 코딩에서 반복되는 승인·테스트·리뷰 동작이 물리 키로 고정되면 사용 습관도 달라집니다. 다만 현재 공개된 것은 티저이므로 구체적인 액션과 지원 IDE는 정식 발표 뒤 확인해야 합니다.

OpenAI가 Codex 사용자를 위한 전용 하드웨어 Codex Micro를 예고했습니다. OpenAI Developers 계정에 올라온 티저에는 “Your favorite Codex shortcuts are getting an upgrade”라는 문구와 함께 작은 매크로패드 형태의 장치가 등장했으며, 정식 공개일은 2026년 7월 15일로 표시됐습니다.

이 제품은 일반 키보드라기보다는 Codex용 단축키 컨트롤러에 가까운 장치로 보입니다. 티저에 등장한 실루엣은 키보드·매크로패드 제조사 Work Louder의 Creator Micro 2와 유사하며, OpenAI와 Work Louder의 로고가 함께 노출됐습니다.

흥미로운 점

Codex Micro는 “AI 코딩 도구를 위한 물리적 인터페이스”라는 점에서 의미가 있습니다. 지금까지 AI 코딩 에이전트는 IDE, 터미널, 웹 UI 안에서만 주로 사용됐지만, OpenAI가 자주 쓰는 Codex 액션을 별도 하드웨어 버튼으로 분리하려는 시도를 하고 있습니다.

개발자 입장에서는 단순한 굿즈가 아니라, Codex 호출·작업 승인·리팩터링·테스트 실행·PR 생성 같은 반복 작업을 물리 버튼에 매핑하는 장치가 될 가능성이 있습니다. 실제로 어느 정도까지 커스터마이즈가 가능한지, VS Code/Cursor/JetBrains 같은 IDE와 어떻게 연동되는지, 표준 HID 장치로 동작하는지는 아직 공개되지 않았습니다.
