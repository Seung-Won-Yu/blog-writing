databow - ADBC로 데이터베이스를 조회하는 CLI 도구

ADBC(Arrow Database Connectivity)를 통해 데이터베이스에 연결하고 SQL을 실행하는 커맨드라인 도구입니다.

호환되는 ADBC 드라이버가 있는 모든 데이터베이스에 연결할 수 있습니다.

명령 히스토리와 직관적인 네비게이션을 지원하는 대화형 SQL 셸을 제공하며, SQL 쿼리 하이라이팅 기능도 있습니다.

동적인 컬럼 너비로 깔끔하게 정렬된 테이블 형태로 결과를 표시하고, 쿼리 결과를 JSON, CSV, Arrow IPC 파일로 익스포트할 수 있습니다.

대화형 사용 외에도 `--query`로 쿼리를 직접 실행하거나, stdin 입력 또는 `--file`로 파일을 실행하고 `--output`으로 결과 파일을 저장하는 등 비대화형 사용도 지원합니다.

Apache-2.0 라이선스로 Rust로 구현되었습니다.

ADBC(Arrow Database Connectivity)는 Apache Arrow 프로젝트의 하위 표준으로, 서로 다른 데이터베이스에서 Arrow 데이터를 넣고 빼기 위한 단일 API입니다.

JDBC/ODBC의 컬럼 지향 대안으로, 결과를 행(row) 단위가 아닌 컬럼 단위 Arrow 데이터로 반환합니다. 이는 대량 컬럼 분석 워크플로우에서 행 지향 형식으로 변환했다가 되돌리는 비용을 피할 수 있어 ODBC/JDBC 대비 효율적입니다.

2023년에 1.0.0이 발표되었고 현재 버전은 1.1.0입니다.

지원하는 데이터베이스(드라이버)로는 공식적으로 PostgreSQL, SQLite, DuckDB, Snowflake, BigQuery, Flight SQL 지원 DB가 있으며, ADBC Driver Foundry 확장을 통해 Amazon Redshift, Apache DataFusion, Apache Spark, ClickHouse, Databricks, Exasol, Microsoft SQL Server도 지원합니다.

오늘의 정처기 문제

운영체제에서 메모리 단편화(Fragmentation) 문제를 해결하기 위한 기법으로 가장 거리가 먼 것은?

  1. 페이징(Paging)
  2. 세그멘테이션(Segmentation)
  3. 메모리 압축(Compaction)
  4. 정답 · 가상 메모리(Virtual Memory)

해설 가상 메모리는 실제 물리 메모리보다 큰 메모리 공간을 사용할 수 있도록 하는 기법입니다. 이는 메모리 단편화 문제 해결보다는 메모리 확장 및 효율적인 사용에 중점을 둡니다. 페이징, 세그멘테이션, 메모리 압축은 모두 단편화 문제를 완화하거나 해결하는 데 기여하는 기법들입니다.
