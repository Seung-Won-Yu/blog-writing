OpenAI GPT 5.6 출시

OpenAI가 2026년 7월 9일, GPT-5.6 패밀리를 정식 출시(GA)했습니다. 6월 26일 프리뷰 이후의 정식 공개이며, Sol(플래그십), Terra(균형형), Luna(최저비용) 세 개의 티어로 구성됩니다.

핵심 요약

효율성 우선, 필요시 최대 성능: Sol은 코딩·지식노동·사이버보안·과학 전 영역에서 SOTA를 달성하면서도 이전/경쟁 모델 대비 더 적은 토큰과 낮은 비용으로 동일 이상의 결과를 냅니다. "성능 대비 비용"이 이번 발표의 핵심 메시지입니다.

ultra 모드 신설: 여러 에이전트를 병렬로 조율해 복잡한 작업을 더 빠르게 처리하는 최고 성능 설정입니다. Responses API에서 멀티에이전트 베타로 사용 가능합니다.

코딩: Artificial Analysis Coding Agent Index 80점으로 SOTA를 달성했습니다 (Claude Fable 5 대비 +2.8점, 토큰 사용량은 절반 이하).

디자인 & 컴퓨터 사용: 렌더링 결과를 직접 검사·수정하는 능력이 강화되었습니다. 프레젠테이션/문서/스프레드시트 품질 향상, 템플릿/참조 문서의 디자인 시스템을 추론해 새 콘텐츠에 일관 적용합니다.

사이버보안: ExploitBench, ExploitGym, SEC-Bench Pro 등에서 GPT-5.5 대비 대폭 향상되었습니다. 다만 자율 공격 실행보다는 취약점 탐지·패치(방어) 쪽에 더 강합니다. Trusted Access for Cyber 프로그램으로 접근을 제어합니다.

과학: 생명과학·화학 벤치마크에서 GPT-5.5 대비 개선되었습니다. (참고: Claude Fable 5는 고급 생물학 질문 대부분을 거부하여 해당 비교에서 제외됨)

내부 연구 가속: OpenAI 내부 연구원의 토큰 사용량이 GPT-5.5 대비 2배 이상 증가했으며, RSI(자기개선) 관련 지표가 16.2점 향상되었습니다.

안전성: 생물·사이버 모두 여전히 'Critical' 임계값 미만입니다. 약 70만 A100e GPU 시간 규모의 레드팀 테스트를 거쳤습니다.

가격 및 제공

ChatGPT, Codex, API 전반에 순차 롤아웃 시작 (24시간 내 전체 적용 예정)

API 가격 (1M 토큰당): Sol: $5 입력 / $30 출력, Terra: $2.5 입력 / $15 출력, Luna: $1 입력 / $6 출력. 프롬프트 캐싱 개선: 명시적 캐시 브레이크포인트, 최소 캐시 수명 30분 도입.

Chatto, 이제 오픈소스로 공개되어 셀프 호스팅 가능

Chatto는 그룹 및 팀 채팅 앱으로, Slack, Teams, Discord와 유사하지만 더 작고 빠른 사용자 경험을 목표로 합니다. 실행 파일만으로 서버를 실행하고 자체 프론트엔드를 제공하여 개인 인프라에 비교적 쉽게 배포할 수 있습니다. 각 서버는 하나의 커뮤니티를 담당하며, 서버 간 연동이나 제3자 추적, 분석 기능은 포함하지 않습니다. 개인 및 채팅 데이터는 사용자별 키로 저장 시 암호화되며, 계정 삭제 시 해당 키는 폐기됩니다. 지난 1년간 개발된 이 애플리케이션은 이제 오픈소스로 공개되어 누구나 직접 운영할 수 있습니다.

셀프 호스팅으로 공개된 Chatto

Chatto는 지난 약 1년간 개발된 그룹 및 팀 채팅 애플리케이션으로, 공식적으로 오픈소스가 되어 누구나 직접 운영할 수 있습니다. 가장 빠른 실행 방법은 Homebrew를 사용하는 것입니다. 설치 절차는 Getting Started Guide에서 확인할 수 있습니다. Slack, Teams, Discord와 같은 그룹 채팅 앱과 비슷한 용도를 목표로 하면서도, 더 컴팩트하고 빠른 사용감을 강조합니다. 실제 동작은 Chatto HQ Community에서 확인할 수 있습니다.

서버 구조와 보안 모델

가장 기본적인 형태에서는 실행 파일 하나로 서버를 실행할 수 있으며, 서버가 자체 프론트엔드도 함께 제공합니다. 리소스 사용량이 낮으며, 개인 및 채팅 데이터는 사용자별 키로 저장 시 완전히 암호화됩니다. 사용자가 계정을 삭제하면 해당 사용자 키가 폐기됩니다. 각 Chatto 서버는 단일 커뮤니티를 구동하며, 서버 간 데이터 연동은 없고 제3자 추적이나 분석도 없습니다. 여러 서버에 동시에 머물고 싶으면 클라이언트가 각 서버에 직접 연결해야 합니다. 여러 커뮤니티를 호스팅하려면 Chatto 프로세스를 여러 개 실행하면 됩니다. 음성 및 영상 통화와 화면 공유를 기본 지원하며, 통화는 종단 간 암호화됩니다. 참가자 수 확장은 사용자의 인프라가 감당할 수 있는 범위까지 가능합니다.

배포 파일과 운영 자료

바이너리는 Linux x86_64·ARM64, macOS, Windows용으로 제공됩니다. 셀프 호스팅 문서는 Chatto Self-Hosting Documentation에서 제공됩니다. 관련 자료로는 Chatto HQ Community(#self-hosting 지원 채널 제공), GitHub Repository, Chatto on Bluesky가 있습니다.

Chatto Cloud와 유료 호스팅

Chatto Cloud는 곧 공개 베타에 들어갈 예정이며, Chatto 서버 유료 호스팅만 제공합니다. 프리미엄 구독, 광고, 기타 추가 요소는 없으며 호스팅만 제공합니다. 출시 시점의 인프라는 유럽 및 유럽 소유 인프라로 구성되며, 더 많은 리전은 2027년 초 출시 예정입니다. 모든 Chatto Cloud 서버는 자동 스케일링, 매일 밤 전체 데이터 백업, 무중단 버전 업그레이드를 제공합니다. Chatto Cloud는 락인을 만들지 않는 방향으로, Chatto Cloud에서 호스팅되는 서버는 셀프 호스팅 서버와 100% 호환됩니다. 언제든 데이터를 옮겨 Chatto Cloud 안팎으로 이동할 수 있습니다.

0.4 상태와 1.0.0까지의 계획

현재 Chatto 버전은 0.4이며, 프로덕션 사용에 충분히 안정적이라고 간주되지만 아직 몇 가지 중요한 기능은 빠져 있습니다. 전체 계획은 Chatto Roadmap에서 볼 수 있습니다. Chatto 0.5의 초점은 안전 기능과 클라이언트 개선이며, 콘텐츠 신고와 모더레이션 기능이 포함됩니다. 특히 멀티 서버 기능을 다듬는 데 집중합니다. Chatto 1.0.0은 약 6~12개월 뒤 도달할 것으로 예상되며, 그 전까지는 파괴적 변경이 있을 수 있습니다. 셀프 호스팅 사용자는 새 버전이 출시될 때 업데이트할 준비가 필요합니다. 릴리스 소식이나 Chatto Cloud 베타 시작 알림은 Chatto announcements 뉴스레터로 받을 수 있으며, 발송 빈도는 약 월 1회입니다.

TypeScript 7.0 발표

TypeScript 7.0은 도구체인을 Go 기반 네이티브 코드로 이식한 릴리스로, 대형 프로젝트의 전체 빌드에서 보통 8~12배 속도 향상을 제공합니다.

새로운 `tsc`는 기존처럼 npm의 `typescript` 패키지로 설치하며, 편집기 지원은 LSP 기반 언어 서버로 VS Code, Visual Studio, WebStorm 등에서 동작합니다.

`vscode`, `sentry`, `bluesky`, `playwright`, `tldraw` 같은 코드베이스에서 빌드 시간이 크게 줄었고, 일부 환경에서는 총 메모리 사용량도 낮아졌습니다.

TypeScript 7.0은 아직 안정적인 프로그래밍 API가 없어 `typescript-eslint`, Vue, MDX, Astro, Svelte, Angular 템플릿 타입 검사 등 TypeScript를 내장하는 도구는 TypeScript 6.0 병행이 필요합니다.

새 기본값과 제거된 옵션이 있어 TypeScript 6.0 변경 사항을 먼저 적용하는 편이 전환에 유리하며, TypeScript 7.1에서 새 API 제공이 예정되어 있습니다.

Go 기반 네이티브 TypeScript

TypeScript 7.0은 TypeScript 도구체인의 네이티브 포트로, Go로 작성된 새 코드베이스를 사용합니다. 기존 코드베이스의 구조와 로직을 최대한 유지하여 두 컴파일러 간 결과의 일관성과 호환성을 맞추는 방향으로 포팅되었습니다.

새 코드베이스는 네이티브 코드 속도, 공유 메모리 기반 멀티스레딩, 여러 최적화를 통해 전체 빌드에서 보통 8~12배 빠른 결과를 냅니다. 설치는 기존처럼 npm으로 가능합니다.

개발 피드백 루프 단축

TypeScript는 편집기 로딩, 전체 참조 찾기, 자동 완성, 진단, `tsc`, `--watch` 모드 등 개발 과정 여러 단계에 관여합니다. TypeScript 7은 이 과정 전반의 대기 시간을 줄여 피드백 루프를 짧게 만듭니다.

VS Code 코드베이스에서 오류가 있는 파일을 열었을 때 첫 오류가 보이기까지 걸리는 시간은 약 17.5초에서 1.3초 미만으로 줄어 13배 이상 빨라졌습니다. 편집기 지원은 LSP 기반이며, 새 언어 서버는 멀티스레딩을 활용해 동시 요청을 빠르게 처리합니다.

VS Code 사용자는 TypeScript 7 전용 확장을 설치할 수 있고, 향후 몇 주 안에 VS Code 자체에도 TypeScript 7 지원이 포함될 예정입니다. Visual Studio 최신 버전은 워크스페이스에 따라 TypeScript 7을 자동 활성화합니다.

오늘의 정처기 문제

관계형 데이터베이스에서 여러 테이블의 데이터를 연결하여 하나의 결과 집합으로 만드는 연산은?

  1. SELECT
  2. INSERT
  3. 정답 · JOIN
  4. UPDATE

해설 JOIN 연산은 관계형 데이터베이스에서 두 개 이상의 테이블을 공통된 컬럼(속성)을 기준으로 연결하여 하나의 결과 테이블로 만드는 데 사용됩니다. 이를 통해 여러 테이블에 분산된 데이터를 통합하여 조회할 수 있습니다. 예를 들어, 주문 정보와 고객 정보를 연결하여 특정 고객의 주문 내역을 조회할 수 있습니다.
