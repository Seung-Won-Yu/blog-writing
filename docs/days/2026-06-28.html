#네트워크

다음 중 TCP(Transmission Control Protocol)의 특징으로 가장 적절한 것은?

  1. ①비연결형(connectionless) 서비스를 제공하여 오버헤드가 적다.
  2. ②데이터 전송의 신뢰성을 보장하지 않으며 속도를 우선한다.
  3. ③흐름 제어(Flow Control)와 혼잡 제어(Congestion Control) 기능을 제공한다.
  4. ④데이터그램(Datagram) 단위로 독립적인 경로를 통해 데이터를 전송한다.

보기를 클릭해 정답을 확인하세요.

정답: ③

TCP는 연결형(connection-oriented) 프로토콜로, 송수신 간의 연결을 설정한 후 데이터를 전송합니다. 이 과정에서 데이터의 신뢰성, 흐름 제어, 혼잡 제어 기능을 제공하여 안정적인 데이터 전송을 보장합니다. UDP는 비연결형 프로토콜로 속도를 우선합니다.

IT · 개발 · 기획 용어
  • 개발 CQRS (Command Query Responsibility Segregation) 데이터 읽기(Query) 작업과 쓰기(Command) 작업을 분리하여 처리하는 아키텍처 패턴으로, 각 작업의 확장성과 성능을 최적화할 수 있습니다.
  • 개발 도메인 주도 설계 (Domain-Driven Design, DDD) 소프트웨어 개발 시 비즈니스 도메인(영역)을 핵심으로 놓고, 도메인 전문가와 개발자가 함께 도메인 모델을 구축하며 문제를 해결해 나가는 접근 방식입니다.
  • 기획 피봇 (Pivot) 기존 사업 모델이나 제품의 방향을 근본적으로 전환하는 전략으로, 시장 변화나 고객 피드백에 맞춰 새로운 기회를 모색할 때 사용됩니다.
