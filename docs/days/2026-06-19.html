#AI 활용

AI 코드 리뷰는 '범위를 좁힌' 프롬프트가 핵심

diff를 통째로 붙이고 '잘 봐줘'라고 하면 칭찬만 돌아온다. 무엇을 볼지 못박으면 진짜 문제를 짚는다.

  1. git diff 를 붙이고 역할을 지정: '시니어 리뷰어로서 버그·보안·엣지케이스만 지적'
  2. 출력 형식 강제: '파일:라인 — 문제 — 수정안' 한 줄씩, 칭찬 금지
  3. 확신도 낮은 항목은 [추측] 표시 요청 → 환각/과잉지적 걸러내기
  4. 지적마다 '왜 문제인지' 한 줄 근거를 같이 받기
