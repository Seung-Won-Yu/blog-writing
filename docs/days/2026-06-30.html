#데이터베이스

데이터베이스 이상(Anomaly) 현상 중, 테이블에서 특정 튜플(행)을 삭제할 때, 그 튜플에만 존재하는 유일한 정보까지 함께 삭제되어 정보 손실이 발생하는 현상은?

  1. ①삽입 이상(Insertion Anomaly)
  2. ②갱신 이상(Update Anomaly)
  3. ③삭제 이상(Deletion Anomaly)
  4. ④조회 이상(Retrieval Anomaly)

보기를 클릭해 정답을 확인하세요.

정답: ③

삭제 이상은 테이블에서 특정 튜플을 삭제할 때, 그 튜플에만 존재하는 다른 유용한 정보까지 함께 삭제되어 정보 손실이 발생하는 문제입니다. 이는 데이터베이스 정규화를 통해 중복을 제거하고 각 테이블의 독립성을 높여 방지할 수 있습니다. 예를 들어, 특정 학생의 수강 과목을 삭제했는데 그 학생에 대한 정보까지 사라지는 경우입니다.

IT · 개발 · 기획 용어
  • 개발 도커 (Docker) 애플리케이션과 그 실행에 필요한 모든 환경(코드, 런타임, 시스템 도구, 라이브러리 등)을 컨테이너라는 격리된 공간에 담아, 어떤 환경에서든 일관되게 실행될 수 있도록 하는 오픈소스 플랫폼입니다.
  • IT RESTful API (Representational State Transfer API) 웹 서비스 디자인 아키텍처 중 하나로, HTTP 메서드(GET, POST, PUT, DELETE)를 사용하여 리소스(자원)에 접근하고 조작하는 방식의 API를 의미합니다.
  • 기획 페어와이즈 테스트 (Pairwise Testing) 테스트 케이스 수를 줄이면서도 높은 결함 발견율을 유지하기 위해, 모든 가능한 매개변수 값 쌍을 한 번 이상 조합하여 테스트하는 기법입니다.
