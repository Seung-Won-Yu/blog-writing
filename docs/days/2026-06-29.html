#소프트웨어 공학

소프트웨어 개발 과정에서 발생할 수 있는 오류를 미리 예측하고 예방하기 위해, 설계 단계에서부터 보안을 고려하여 개발하는 접근 방식은?

  1. ①퍼징(Fuzzing)
  2. ②보안 코딩(Secure Coding)
  3. ③시큐어 SDLC(Secure SDLC)
  4. ④페어 프로그래밍(Pair Programming)

보기를 클릭해 정답을 확인하세요.

정답: ③

시큐어 SDLC(Secure Software Development Life Cycle)는 소프트웨어 개발 생명 주기 전반에 걸쳐 보안 활동을 통합하여 보안 취약점을 최소화하는 접근 방식입니다. 이는 설계부터 구현, 테스트, 배포, 유지보수까지 모든 단계에서 보안을 고려합니다. 퍼징은 테스트 기법이고, 보안 코딩은 구현 단계의 활동이며, 페어 프로그래밍은 개발 방식입니다.

03
IT · 개발 · 기획 용어
  • 개발 가비지 컬렉션 (Garbage Collection) 프로그램에서 더 이상 사용되지 않는 메모리(가비지)를 자동으로 찾아 해제하여 메모리 누수를 방지하고 효율적인 메모리 관리를 돕는 기능입니다.
  • 개발 싱글톤 패턴 (Singleton Pattern) 객체 지향 디자인 패턴 중 하나로, 특정 클래스의 인스턴스가 오직 하나만 생성되도록 보장하고, 이 인스턴스에 대한 전역적인 접근점을 제공하는 패턴입니다.
  • 기획 퍼널 분석 (Funnel Analysis) 사용자가 서비스나 제품에서 특정 목표를 달성하기까지 거치는 일련의 단계를 시각화하고, 각 단계에서 이탈하는 사용자 수를 분석하여 개선점을 찾는 방법입니다.
