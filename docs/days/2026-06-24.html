#git

git 실수, 일단 reflog부터 — 거의 다 되돌린다

잘못 reset·rebase·삭제해 커밋이 사라진 듯해도, git reflog에 HEAD 이동 기록이 남아 대부분 복구된다.

  1. git reflog 로 최근 HEAD 이동 이력과 각 시점 커밋 해시 확인
  2. 되돌릴 시점 해시(예: HEAD@{3})를 고른다
  3. git reset --hard <해시> 로 복원, 또는 git checkout -b rescue <해시> 로 안전하게 분기
  4. 기본 90일 보관이니 사고 직후 바로 확인
