# 티스토리 디자인 시스템

이 디렉터리가 블로그 디자인의 단일 기준이다. 스킨과 게시물 생성기가 같은 요소를 각각 꾸미지 않도록 소유권을 나눈다.

## 방향

컨셉은 **개발자의 아침 브리핑 + 검증 노트**다. 화려한 랜딩페이지보다 빠른 탐색, 편안한 본문, 분명한 출처와 운영자의 관점을 우선한다.

홈은 최신 글 한 편을 편집부의 리드 기사처럼 크게 보여주고, 이전 글은 2열 아카이브로 이어진다. 브랜드명에만 차분한 세리프를 쓰고 제목·본문·메타 정보는 산세리프로 통일한다. 색은 먹색·종이색·짙은 초록 한 가지 강조색만 사용하며 그라데이션, 떠 있는 그림자, 둥근 배지 남용을 금지한다.

- GeekNews에서 가져올 원칙: 출처·시간·주제를 짧고 빠르게 훑는 정보 밀도
- 요즘IT에서 가져올 원칙: 제목·이미지·예상 읽기 시간으로 진입 장벽 낮추기
- The Pragmatic Engineer에서 가져올 원칙: 한 명의 편집 관점, 일관된 장문 형식, 인기 글 아카이브
- 가져오지 않을 것: 복잡한 상단 메뉴, 과도한 카드·배지, 서비스형 랜딩페이지 느낌

## 소유권

| 영역 | 소유 파일 | 규칙 |
|---|---|---|
| 전체 스킨 HTML | `skin.html` | 티스토리 HTML 편집기의 단일 원본 |
| 홈·사이드바 HTML 조각 | `skin-components.html` | `skin.html`에서 찾기 쉬운 참조용 조각 |
| 헤더·홈·목록·사이드바·글 제목 | `skin-layer.css` | 스킨만 스타일링 |
| 뉴스 본문 구조 | `blog_pipeline/publishing/export_tistory.py` | 클래스와 의미 구조만 출력 |
| 뉴스 본문 시각 디자인 | `skin-layer.css`의 `.daily-digest-post` 구역 | 반드시 이 네임스페이스 안에서만 지정 |
| 초안 미리보기 | `build_copy_page.py` | 같은 `skin-layer.css`를 읽어 렌더링 |
| 광고 | 티스토리 revenue 태그 | NEWS 01과 NEWS 02 사이의 최상위 형제 요소 |

게시물 생성기는 인라인 스타일을 출력하지 않는다. 티스토리 기본 규칙과 충돌할 수 있는 선택자는 `.entry-content > .daily-digest-post` 아래로 제한한다. 모든 하위 요소에 `box-sizing: border-box`를 강제해 폭과 패딩이 겹치지 않게 한다.

## 게시물 정보 구조

1. 티스토리 스킨 제목(H1) — 본문에서 제목을 반복하지 않음
2. 날짜·예상 읽기 시간과 도입 문단
3. 대표 이미지
4. 세 뉴스를 잇는 한 문단의 관점
5. `이 글에서 볼 것` — 기사 제목 3개만 표시
6. 뉴스 3개 — 이미지, 출처, 제목, 요약, 본문, 원문 링크
7. NEWS 01 뒤 AdFit 광고 1개
8. 정처기 문제, 용어, 마무리 행동

`먼저 보는 핵심`과 `3분 미리보기`처럼 같은 제목을 두 번 나열하지 않는다. 뉴스 본문은 카드 안에 다시 패딩을 넣지 않고 760px 읽기 폭을 그대로 사용한다.

## 티스토리 적용

1. 관리자 → 꾸미기 → 스킨 편집 → CSS로 이동한다.
2. 기존 CSS에서 `Tistory skin custom layer` 주석부터 파일 끝까지 삭제한다.
3. `skin-layer.css` 전체를 붙여넣는다.
4. HTML 전체를 바꿀 때는 `skin.html`을 사용한다. 부분 수정만 할 때는 `dev-hero`, `dev-profile-card` 블록을 `skin-components.html`과 대조한다.
5. 저장 전 현재 HTML/CSS를 별도 백업한다.

기본 스킨 CSS는 건드리지 않고 마지막 사용자 레이어만 교체한다. 이후 디자인 변경도 이 파일 한 곳에서만 한다.

## 성장 지표와 운영 원칙

- 유입: 질문형·구체형 제목, 한 글 한 검색 의도
- 체류: 넓은 본문, 중복 없는 목차, 기사마다 구체적 사례와 확인 방법
- 재방문: 매일 같은 시간과 형식, 주 1회 직접 써본 회고/가이드
- 신뢰: 원문 링크, 사실과 해석 분리, 자동 요약에 없는 직접 확인 또는 비교
- 내부 이동: 관련 글 2~3개를 글 하단에 연결

Google은 검색 유입만 노린 대량 자동 요약보다 독자에게 실제로 도움이 되는 독창적 분석, 충분한 설명, 직접 경험과 명확한 사이트 목적을 권장한다. 따라서 뉴스 3개를 늘리는 것보다 “왜 중요한지, 무엇을 직접 확인했는지”를 유지하는 편이 우선이다.
