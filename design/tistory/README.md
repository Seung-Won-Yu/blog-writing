# 티스토리 디자인 시스템

이 디렉터리가 블로그 디자인의 단일 기준이다. 스킨과 게시물 생성기가 같은 요소를 각각 꾸미지 않도록 소유권을 나눈다.

사이트별 비교와 채택·제외 기준은 `BENCHMARK.md`를 따른다.

## 방향

컨셉은 **조용한 개인 기술 저널**이다. 화려한 랜딩페이지보다 빠른 탐색, 편안한 장문, 짧은 출처와 운영자의 확인 과정을 우선한다. 목록에는 카드가 있지만 글 안에는 카드가 없다.

홈은 별도의 선언형 소개 배너 없이 최신 글부터 보여준다. 한 페이지의 글 6개를 같은 크기의 2열 아카이브로 보여주고 모바일에서는 1열로 전환한다. 첫 글만 과도하게 키우지 않아 제목·요약·날짜가 항상 같은 위치에서 보이고 마지막 줄도 비대칭으로 남지 않게 한다. 제목·본문·메타 정보는 산세리프로 통일한다. 색은 흰색·먹색·짙은 초록 한 가지 강조색만 사용하며 그라데이션, 떠 있는 그림자, 둥근 배지 남용을 금지한다.

보조 페이지도 같은 편집 체계를 쓴다. 태그는 200개를 작은 배지로 흩뿌리지 않고 3열 색인으로 보여주며, 방명록은 제목과 용도를 먼저 안내한다. 사이드바는 블로그 소개·분류·공지·최근/인기 글을 중심으로 두고, 방문자 수는 가장 아래에 작은 통계 블록으로 표시한다. 무작위 태그와 최근 댓글처럼 글 읽기를 돕지 않는 위젯은 제거한다. 푸터는 기본 스킨 제작 문구 대신 블로그 이름과 운영 문장으로 닫는다.

- GeekNews에서 가져올 원칙: 출처·시간·주제를 짧고 빠르게 훑는 정보 밀도
- 요즘IT에서 가져올 원칙: 제목·이미지·예상 읽기 시간으로 진입 장벽 낮추기
- The Pragmatic Engineer에서 가져올 원칙: 한 명의 편집 관점, 일관된 장문 형식, 인기 글 아카이브
- 가져오지 않을 것: 복잡한 상단 메뉴, 과도한 카드·배지, 서비스형 랜딩페이지 느낌

## 소유권

| 영역 | 소유 파일 | 규칙 |
|---|---|---|
| 전체 스킨 HTML | `skin.html` | 티스토리 HTML 편집기의 단일 원본 |
| 전체 스킨 CSS | `style.css` | 기본 스킨 CSS와 사용자 레이어를 합친 안전한 전체 교체본 |
| 홈·사이드바 HTML 조각 | `skin-components.html` | `skin.html`에서 찾기 쉬운 참조용 조각 |
| 헤더·홈·목록·사이드바·글 제목 | `skin-layer.css` | `style.css` 끝에 한 번만 들어가는 사용자 레이어 |
| 뉴스 본문 구조 | `blog_pipeline/publishing/export_tistory.py` | 클래스와 의미 구조만 출력 |
| 뉴스 본문 시각 디자인 | `skin-layer.css`의 `.daily-digest-post` 구역 | 반드시 이 네임스페이스 안에서만 지정 |
| 초안 미리보기 | `build_copy_page.py` | 같은 `skin-layer.css`를 읽어 렌더링 |
| 광고 | 티스토리 revenue 태그 | NEWS 01과 NEWS 02 사이의 최상위 형제 요소 |

게시물 생성기는 인라인 스타일을 출력하지 않는다. 티스토리 기본 규칙과 충돌할 수 있는 선택자는 `.entry-content > .daily-digest-post` 아래로 제한한다. 모든 하위 요소에 `box-sizing: border-box`를 강제해 폭과 패딩이 겹치지 않게 한다.

## 게시물 정보 구조

1. 티스토리 스킨 제목(H1) — 본문에서 제목을 반복하지 않음
2. 날짜·예상 읽기 시간과 도입 문단
3. 대표 이미지
4. 세 뉴스를 잇는 한 문단의 관점
5. `글 순서` — 기사 제목 3개만 표시
6. 뉴스 3개 — 이미지, 출처, 제목, 요약, 본문, 원문 링크
7. NEWS 01 뒤 AdFit 광고 1개
8. 정처기 문제, 용어, 마무리 행동

`먼저 보는 핵심`과 `3분 미리보기`처럼 같은 제목을 두 번 나열하지 않는다. 뉴스 본문은 카드 안에 다시 패딩을 넣지 않고 720px 읽기 폭을 그대로 사용한다. 날짜·예상 읽기 시간 아래의 자동화 고지나 구성 설명 문장은 출력하지 않는다.

## 티스토리 적용

1. 관리자 → 꾸미기 → 스킨 편집 → CSS로 이동한다.
2. CSS 편집기 전체를 교체할 때는 반드시 `style.css` 전체를 붙여넣는다.
3. `skin-layer.css`는 기본 CSS가 이미 남아 있을 때 사용자 레이어만 교체하는 파일이다. 이 파일만 CSS 편집기 전체에 붙여넣으면 모바일·목록 레이아웃이 깨진다.
4. 사용자 레이어만 갱신할 때는 `won0322.tistory.com · canonical custom layer` 주석부터 파일 끝까지만 `skin-layer.css`로 교체한다.
5. HTML 전체를 바꿀 때는 `skin.html`을 사용한다. 부분 수정만 할 때는 `dev-profile-card` 블록을 `skin-components.html`과 대조한다.
6. 저장 전 현재 HTML/CSS를 별도 백업한다.

`style.css`에는 기본 CSS가 정확히 한 번, `skin-layer.css` 내용도 정확히 한 번 있어야 한다. 이후 디자인 변경은 먼저 `skin-layer.css`에서 하고, 같은 레이어를 `style.css` 끝에 반영한다.

## 성장 지표와 운영 원칙

- 유입: 질문형·구체형 제목, 한 글 한 검색 의도
- 체류: 넓은 본문, 중복 없는 목차, 기사마다 구체적 사례와 확인 방법
- 재방문: 매일 같은 시간과 형식, 주 1회 직접 써본 회고/가이드
- 신뢰: 원문 링크, 사실과 해석 분리, 자동 요약에 없는 직접 확인 또는 비교
- 내부 이동: 관련 글 2~3개를 글 하단에 연결

Google은 검색 유입만 노린 대량 자동 요약보다 독자에게 실제로 도움이 되는 독창적 분석, 충분한 설명, 직접 경험과 명확한 사이트 목적을 권장한다. 따라서 뉴스 3개를 늘리는 것보다 “왜 중요한지, 무엇을 직접 확인했는지”를 유지하는 편이 우선이다.
