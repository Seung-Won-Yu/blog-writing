# 티스토리 디자인 벤치마크와 결정

## 결론

이 블로그의 포지션은 **회사 기술 미디어**가 아니라 **한 명의 개발자가 매일 읽고 확인한 것을 축적하는 개인 기술 저널**이다.

홈은 카드형 아카이브를 유지하되 글 안에서는 카드를 반복하지 않는다. GeekNews의 짧은 출처 표기, 44BITS의 흰 바탕 장문 폭, 요즘IT의 제목·메타 계층, AI타임스의 뉴스형 헤드라인을 조합한다. Toss Tech는 완성도와 여백만 참고하고 대형 브랜드 히어로·프로모션 구조는 가져오지 않는다.

## 비교

| 기준 | 강점 | 이 블로그에 적용 | 적용하지 않음 |
|---|---|---|---|
| Toss Tech | 높은 이미지 품질, 명확한 추천 글, 저자와 시리즈 | 제목의 구체성, 대표 이미지의 일관된 품질 | 대형 캐러셀, 기업 채용 CTA, 전용 일러스트 전제 구조 |
| 우아한형제들 기술블로그 | 강한 브랜드 인지, 폭넓은 기술 분류 | 기술 분야를 찾기 쉬운 카테고리 | 대형 브랜드 배너, 많은 태그를 홈 전면에 노출 |
| 요즘IT | 카테고리·읽기 시간·인기 순위로 빠른 탐색 | 예상 읽기 시간, 짧은 요약, 모바일 카드 가독성 | 포털형 다단 메뉴, 랭킹·커뮤니티 기능, 카드 종류 혼합 |
| GeekNews | 높은 정보 밀도, 원문 우선, 빠른 스캔 | 원문 링크, 사실과 해석 분리, 매일 읽기 좋은 밀도 | 투표·댓글 중심 뉴스 보드 UI |
| 44BITS | 흰 바탕, 약 720~760px 장문 폭, 타이포그래피 중심 | 본문 카드 제거, 충분한 행간, 단순한 구분선 | 기술 문서형 긴 목차를 모든 글에 강제 |
| AI타임스 | 기사 제목과 리드가 빠르게 읽히는 뉴스 계층 | 구체적인 제목, 짧은 출처·날짜, 본문 우선 | 포털형 광고 밀도와 복잡한 우측 영역 |
| The Pragmatic Engineer | 글과 저자 중심, 절제된 홈, 장기 신뢰 | 최신 글 우선, 단순한 목록, 한 명의 일관된 관점 | 영어 뉴스레터형 사이드 메뉴의 그대로 복제 |

## 홈 규칙

1. 블로그 헤더 다음에 선언문·소개 배너·카테고리 CTA를 반복하지 않는다.
2. 광고 다음에 최신 글 6개를 같은 크기의 2열 카드로 보여준다.
3. 모바일은 같은 6개를 1열로 전환하고 페이지 번호를 유지한다.
4. 첫 글만 크게 만들지 않는다. 모든 글에서 이미지·제목·요약·날짜가 보여야 한다.
5. 홈의 색은 흰색, 먹색, 짙은 초록만 사용한다.
6. 카드 그림자, 그라데이션, 영문 장식 문구, 둥근 배지와 과장된 선언문을 쓰지 않는다.

## 사이트 전체 규칙

1. 헤더·목록·본문·태그·방명록·사이드바·푸터가 같은 색상 토큰과 제목 계층을 사용한다.
2. 사이드바는 글 탐색에 필요한 소개·분류·공지·최근/인기 글을 우선한다. 방문자 수는 가장 아래에 작게 두고, 무작위 태그는 태그 페이지의 역할로 돌린다.
3. 태그 페이지는 배지 구름이 아니라 가나다/영문을 빠르게 훑는 색인처럼 보이게 한다.
4. 방명록과 검색 결과 같은 보조 페이지도 제목과 한 문장 안내를 먼저 보여준다.
5. 글 하단의 태그·관련 글·댓글은 본문과 같은 720px 읽기 폭을 사용한다.
6. 푸터에서 티스토리 기본 제작 문구를 노출하지 않고 블로그의 목적과 운영자 이름을 남긴다.

## 시각 시스템

방향은 **조용한 개인 기술 저널**이다. 목록에서는 탐색을 돕는 카드가 있어도 글 안에서는 흰 바탕, 제목, 본문, 얇은 선만으로 읽게 한다. 강조색은 링크와 번호에만 쓰며 기업 전용 일러스트나 대형 브랜드 배너는 사용하지 않는다.

| 역할 | 색상 | 사용처 |
|---|---|---|
| Canvas | `#FFFFFF` | 전체 배경 |
| Surface | `#FFFFFF` | 카드, 입력창, 본문 표면 |
| Surface Subtle | `#F6F7F5` | 이미지 로딩 면, 아주 약한 보조 배경 |
| Ink | `#171B18` | 글 제목, H2·H3·H4, 핵심 문장 |
| Text | `#343A36` | 본문과 요약 |
| Muted | `#6F7672` | 날짜, 작성자, 보조 정보 |
| Accent | `#17694E` | 분류, 번호, 링크, 포커스 |
| Line | `#E4E7E5` | 구분선과 카드 테두리 |
| Warning | `#9B6A24` | 오늘 해볼 일과 주의 정보 |

- 큰 제목을 초록색으로 칠하지 않는다. 초록은 분류·링크·번호처럼 작은 탐색 신호에만 쓴다.
- 글 제목 영역에는 색 면이나 카드 테두리를 넣지 않는다.
- 본문 소제목은 장식 선이나 배지를 붙이지 않고 먹색과 크기·굵기만으로 계층을 만든다.
- 제목과 본문은 동일한 산세리프 계열을 사용한다.
- 데스크톱 기본 본문은 17px·행간 1.9, 뉴스 제목은 27px, 글 제목은 38px, 읽기 폭은 720px를 기준으로 한다.
- 자동 다크 모드는 지원하지 않는다. 티스토리 편집기·광고·기존 게시물의 고정색 때문에 일부 영역만 어두워지는 불완전한 모드보다 검증된 라이트 테마 하나를 유지한다.

## 글 규칙

1. 제목은 독자가 얻을 답이나 확인할 질문을 구체적으로 적는다.
2. 첫 문단은 뉴스 소개가 아니라 독자가 겪을 장면이나 판단부터 시작한다.
3. 사실, 의미, 직접 확인할 점을 구분하되 편집자 역할을 별도 상자로 과시하지 않는다.
4. 뉴스 이미지는 물건·화면·문서·작업 공간이 중심인 자연스러운 다큐멘터리 사진을 우선한다. 영화 조명, 초현실 합성, 네온, 정면 인물 스톡 사진은 피한다.
5. 원문 링크와 확인 기준을 남기고, 자동화 사용 고지나 운영자의 메모를 본문에 반복하지 않는다.
6. AdFit은 NEWS 01이 완전히 끝난 뒤 NEWS 02 시작 전에 한 번만 둔다.
7. 자동 상·하단 광고, NEWS 01 뒤 수동 AdFit, 사이드바 광고를 유지한다. 광고를 없애지 않고 앞뒤 여백과 본문 기준선만 정돈한다.

## 변경 승인 기준

- 방문자가 첫 화면에서 3초 안에 최신 글의 제목과 내용을 파악할 수 있는가?
- 모바일 390px에서 가로 스크롤 없이 이미지·제목·요약·페이지 번호가 보이는가?
- 스킨 장식이 글보다 먼저 보이거나 더 강하게 보이지 않는가?
- 특정 회사 사이트의 자산이나 규모를 흉내 내지 않고 개인 저널의 신뢰가 드러나는가?
- 동일한 소개, 목차, 제목, 광고가 두 번 출력되지 않는가?
